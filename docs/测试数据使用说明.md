# 测试数据使用说明

> 一组带明确 marker 的虚拟数据，用于回归测试 Sprint 2/3/5 的 UI/UX 改动。所有数据**仅生产数据库**——本地不需要这些（本地用真实账号注册即可）。

## 灌入数据

ECS 上跑（docker cp 拷脚本进 web 容器再 node 执行）：

```bash
ssh root@8.159.155.134
cd ~/piaociyuan
git pull origin main

docker cp apps/web/scripts/seed-test-data.js piaociyuan-web:/tmp/
docker compose exec web node /tmp/seed-test-data.js
```

脚本是 **idempotent** 的——重复跑只会跳过已存在的记录，不会重复创建。

## 清理数据

```bash
docker cp apps/web/scripts/cleanup-test-data.js piaociyuan-web:/tmp/
docker compose exec web node /tmp/cleanup-test-data.js
```

清理脚本只按 marker 匹配，**绝对不会**误删真实数据。

## 注意事项

1. **测试环境隔离**：脚本设计为可重复运行，不污染真实用户数据。所有 marker（phone `139000000XX`、nickname `[TEST]`、event name `[TEST]`、event desc `[TEST_DATA]`）都是不会自然出现的。
2. **测试完务必清理**：留着 5 个测试用户和 4 个测试活动会出现在管理后台、admin 统计、首页排序里。回归测完跑 `cleanup-test-data.js`。
3. **Mobile 测试**：用 Expo Go 扫 Metro 二维码即可登录测试账号；登录态本地存储独立。
4. **生产数据安全**：cleanup 脚本对真实数据**完全无影响**（按 marker 严格匹配）。即便误跑也不会损坏真实用户。
5. **混合对话**（极端情况）：如果某个真实用户跟测试用户 A 也开过对话（理论上不该有人主动私信测试账号），cleanup 脚本会保留对话本身（删除 A 的 participant 即可），不会删真实用户那边的消息历史。

## 故障排查

### `Cannot find module '@prisma/client'`
脚本在 web container 里跑，prisma client 已经在 `/app/node_modules`。如果在 host 上直接 node 跑会报这个错——必须用 `docker compose exec web node ...`。

### `Unique constraint violation`
说明已有数据冲突。先跑 cleanup-test-data.js 清干净再 seed。

### 灌入数据后页面不显示
检查浏览器是否登录了非测试账号；测试帖子的 isVisible=true 默认所有人可见。如果首页 / feed 还是看不到，看看 / signals 列表是否需要刷新（Next.js cache）或检查 admin 后台 → posts 列是否被标 isVisible=false。
