## 触发时机矩阵

| 时机 | 跑什么 | 谁跑 |
|------|--------|------|
| **commit 前** | 本地 `cd apps/web && npm run build` + `cd apps/mobile && npx tsc --noEmit` | 开发者 |

| **CI 绿后** | 本文件"已 ship 的功能测试入口" 中**本次改动相关项** | 开发者 / 测试者 |

| **升依赖 / 改环境后** | 单跑 [一致性测试清单](一致性测试清单.md) | 开发者 |

## 综合 e2e 测试流程（Web 完整跑一遍约 15 分钟）

按顺序操作，覆盖核心 user journey：

1. **登录** 13900000001 / Test123456!（W-F2/F5/A4）
2. **主页** /events → 看 active 高亮 + 卡片显示（W-N1/L1/P1）
3. **宇宙信号** /signals → 切类目 → 进一个活动详情（W-N4 面包屑）
4. **购票** 选数量 → checkout → 顶部返回按钮可见
5. **/encore 安可区** 进帖子 → 点图 lightbox（W-S4）→ 评论分页（W-S3）
6. **/u/[B] 用户主页** → 互关 badge（W-S7）→ 发私信按钮（W-S2）
7. **私信会话** 加载更早消息 → 发文字 → 发图片 → 失败重试
8. **/messages** 列表显示**群聊**（点 + 群聊创建）
9. **/notifications** 通知页 → 标已读
10. **/search** 顶栏搜索 → 综合结果页
11. **/tickets** 票列表 → 详情 → QR → 转赠 → 接收账号验证
12. **/admin/collectibles**（用 13900000099）添加收藏品
13. **退出登录** → ConfirmDialog（W-T4）

## 新功能 ship 后的检查清单

每次 ship 必须：

1. ✅ **本文件加入"已 ship 的功能测试入口"**：在最近一组 batch 表格加新行（改动 ID / 测试入口 / 期望效果）
2. ✅ **如新增 seed 数据**：同步改 `apps/web/scripts/seed-test-data.js` + 更新 [测试数据使用说明.md](测试数据使用说明.md)
3. ✅ **如改 admin 流程**：同步改 [admin-功能测试清单.md](admin-功能测试清单.md)
4. ✅ **如改跨端共享逻辑（同时影响 mobile + web）**：同步改 [Web-Mobile功能对齐审计.md](Web-Mobile功能对齐审计.md)
5. ✅ **如新增冒烟级 P0 流程**：同步改 [冒烟测试清单.md](冒烟测试清单.md)

漏更新算 ship 不完整。下次 session（包括其他 Claude 实例）会通过本文找到该测什么，没列出的功能视为"未交付"。
