## 关键服务说明

### 1. GitHub Actions (CI/CD)

**配置文件**: `.github/workflows/deploy.yml`

**触发条件**: `main` 分支 push 或手动触发

**流程**:
1. **validate** — `npm ci` → `tsc --noEmit` → `next lint`（均 continue-on-error）
2. **build** — 构建 Docker 镜像 → 推送到 `ghcr.io/viosus/piaociyuan/piaoyuzhou-web:latest`
3. **deploy** — SSH 到 ECS → `docker pull` → `docker-compose down/up` → `prisma db push`

**GitHub Secrets 配置**:
| Secret | 说明 |
|--------|------|
| `ECS_HOST` | ECS 公网 IP: `8.159.155.134` |
| `ECS_USERNAME` | SSH 用户名: `root` |
| `ECS_SSH_KEY` | SSH 私钥 |
| `GHCR_TOKEN` | GitHub Container Registry Token |

### 2. 阿里云 ECS

- **系统**: Ubuntu 24.04 LTS
- **公网 IP**: `8.159.155.134`
- **内网 IP**: `172.19.215.12`
- **项目目录**: `~/piaociyuan`
- **SSH 连接**: `ssh root@8.159.155.134`

**docker-compose 服务** (配置文件: `docker-compose.yml`):

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| web | piaoyuzhou-web | 3000 | Next.js + Socket.io 自定义服务器 |
| nginx | piaoyuzhou-nginx | 80, 443 | 反向代理 + SSL |
| redis | piaoyuzhou-redis | 6379 | 缓存 |

**环境变量**: 存储在 `~/piaociyuan/.env`，包含 DATABASE_URL、JWT_SECRET 等

- **实例 ID**: `pgm-uf6aoud8qrmcf922`

## Docker 镜像构建

**Dockerfile**: `apps/web/Dockerfile`

**两阶段构建**:
1. **builder 阶段**: 安装依赖 → `prisma generate` → 构建 shared 包 → `next build`
2. **runner 阶段**: 复制 `.next/`、`public/`、`server.js`、`lib/`、`prisma/`、`node_modules/`

**镜像仓库**: `ghcr.io/viosus/piaociyuan/piaoyuzhou-web`

## 数据库变更流程

Schema 修改后的部署流程:

```
1. 本地修改 apps/web/prisma/schema.prisma
2. git push 到 main 分支
3. CI/CD 自动构建 + 部署
4. 部署脚本自动执行 prisma db push 同步表结构到 RDS
```

**手动操作数据库**:
```bash
# SSH 到 ECS
ssh root@8.159.155.134

# 同步 schema
cd ~/piaociyuan
docker-compose exec web npx prisma db push

# 直接连 RDS 执行 SQL
PGPASSWORD="你的密码" psql -h pgm-uf6aoud8qrmcf922.pg.rds.aliyuncs.com -U piaociyuan -d piaociyuan
```

**注意事项**:
- `prisma db push` 不使用迁移文件，直接将 schema 同步到数据库
- 如果需要迁移历史记录，改用 `prisma migrate deploy`
- 破坏性变更（删列/删表）`prisma db push` 会提示确认

## 常用运维命令

```bash
# SSH 到 ECS
ssh root@8.159.155.134
cd ~/piaociyuan

# 查看容器状态
docker ps
docker-compose logs -f web    # 实时日志
docker-compose logs --tail=100 web  # 最近100行

# 重启服务
docker-compose restart web

# 完整重部署
docker-compose down && docker-compose up -d

# 数据库操作
docker-compose exec web npx prisma db push       # 同步 schema
docker-compose exec web npx prisma studio         # GUI 管理

# 手动部署（不通过 CI/CD）
./deploy.sh build    # 本地构建
./deploy.sh start    # 启动
./deploy.sh migrate  # 执行迁移
./deploy.sh health   # 健康检查
```
