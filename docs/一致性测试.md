# 一致性测试清单

> 本文档用于持续检查项目环境、依赖、配置的一致性。
> 每次大版本更新或架构变更后应重新执行本清单。
> 最后更新: 2026-03-26

---

## 1. 依赖版本一致性

跨 workspace 的共享依赖必须使用相同版本。

### 1.1 检查项：共享依赖版本对比

| 依赖 | 预期版本 | Web (`apps/web/package.json`) | Mobile (`apps/mobile/package.json`) | Shared (`packages/shared/package.json`) |
|------|----------|-------------------------------|--------------------------------------|------------------------------------------|
| react | 19.1.0 | `dependencies` | `dependencies` | - |
| react-dom | 19.1.0 | `dependencies` | - | - |
| react-native | 0.81.5 | - | `dependencies` | - |
| typescript | ~5.9.2 | `devDependencies` | `devDependencies` | `devDependencies` |
| three | ^0.172.0 | `dependencies` | `dependencies` | - |
| socket.io-client | ^4.8.1 | `dependencies` | `dependencies` | - |
| @piaociyuan/shared | * | `dependencies` | `dependencies` | - |

### 1.2 检查项：根 overrides

**文件**: `package.json` (根目录)

```json
"overrides": {
  "react": "19.1.0",
  "react-native": "0.81.5",
  "use-latest-callback": "0.3.3"
}
```

**验证方法**: 确认 overrides 中的版本与各 workspace 的 `dependencies` 声明一致。

| 框架 | 预期版本 | 文件位置 |

| @prisma/client | ^6.18.0 | `apps/web/package.json` |
| prisma (CLI) | ^6.18.0 | `apps/web/package.json` devDependencies |
| @google/model-viewer | ^4.1.0 | `apps/web/package.json` |

## 2. Node.js 版本一致性

Node.js 版本出现在 5 个位置，必须保持一致。

### 2.1 检查项

| 位置 | 文件 | 字段/行 | 预期值 |
|------|------|---------|--------|
| 本地环境 | - | `node -v` | v22.x |
| Dockerfile builder | `apps/web/Dockerfile` 第 2 行 | `FROM node:XX-alpine` | `node:22-alpine` |
| Dockerfile runner | `apps/web/Dockerfile` 第 26 行 | `FROM node:XX-alpine` | `node:22-alpine` |
| GitHub Actions | `.github/workflows/deploy.yml` 第 23 行 | `node-version` | `'22'` |
| package.json engines | `package.json` (根) | `engines.node` | `>=18.0.0` |
| CLAUDE.md | `CLAUDE.md` | 运行环境 Node.js | `>= 18.0.0 (当前 22.x)` |

**验证方法**:
```bash
node -v
grep -n "node:" apps/web/Dockerfile
grep -n "node-version" .github/workflows/deploy.yml
```

## 3. 环境变量一致性

### 3.1 检查项：JWT 配置

| 变量 | 预期值 | 根 `.env.example` | Web `.env.example` | `docker-compose.yml` | `auth.ts` 默认值 | Shared constants |
|------|--------|-------------------|--------------------|-----------------------|-------------------|-------------------|
| JWT_ACCESS_EXPIRES | 15m | `15m` | `15m` | `${JWT_ACCESS_EXPIRES:-1d}` | `1d` (开发回退) | `15m` |
| JWT_REFRESH_EXPIRES | 7d | `7d` | `7d` | `${JWT_REFRESH_EXPIRES:-7d}` | `7d` | `7d` |

> 注意: `auth.ts` 中 Access Token 默认 `1d` 是有意为之的开发便利设置，生产环境通过 `.env` 覆盖为 `15m`。`docker-compose.yml` 的 `1d` 同理，仅作为 fallback。

**文件位置**:
- `apps/web/lib/auth.ts` — `generateAccessToken()` / `generateRefreshToken()`
- `packages/shared/src/constants/index.ts` — `JWT_CONFIG`
- `.env.example` (根)
- `apps/web/.env.example`
- `docker-compose.yml` — web service environment

### 3.2 检查项：必需环境变量清单

**Web 必需** (缺失会导致启动失败):
| 变量 | 用途 | 验证位置 |
|------|------|----------|
| `DATABASE_URL` | PostgreSQL 连接串 | `apps/web/prisma/schema.prisma` |
| `JWT_SECRET` | JWT 签名密钥 | `apps/web/lib/auth.ts` (启动时校验) |
| `ENCRYPTION_KEY` | 敏感数据加密 | `apps/web/lib/` |

**Mobile 必需**:
| 变量 | 用途 | 验证位置 |
|------|------|----------|
| `EXPO_PUBLIC_API_URL` | API 基础地址 | `apps/mobile/src/config/api.ts` |

**验证方法**: 对比 `.env.example` 文件中的变量与代码中 `process.env.XXX` 的引用是否完整。

### 4.1 检查项：TypeScript 配置

| 设置 | Web (`apps/web/tsconfig.json`) | Mobile (`apps/mobile/tsconfig.json`) | Shared (`packages/shared/tsconfig.json`) |
|------|------|--------|--------|
| strict | `true` | `true` | `true` |
| @piaociyuan/shared 别名 | `../../packages/shared/src` | `../../packages/shared/src/index.ts` | - |
| @/* 别名 | `./*` | `./*` | - |

### 4.2 检查项：TypeScript 版本范围

所有 workspace 的 `typescript` devDependency 应统一使用 `~5.9.2`。

**验证方法**:
```bash
grep -r '"typescript"' apps/web/package.json apps/mobile/package.json packages/shared/package.json
```

## 5. 文档一致性

### 5.1 检查项：CLAUDE.md API 路由 vs 实际目录

**验证方法**: 对比 CLAUDE.md 中列出的 API 模块与 `apps/web/app/api/` 下的实际目录。

```bash
ls apps/web/app/api/
```

每个实际存在的目录都应在 CLAUDE.md 中有对应条目。

### 5.2 检查项：部署文档 vs 实际架构

| 文档声明 | 实际状态 | 需检查文件 |
|----------|----------|------------|
| 数据库: 阿里云 RDS | 确认 docker-compose.yml 无 PostgreSQL 容器 | `docker-compose.yml` |
| Docker Registry | `ghcr.io/viosus/piaociyuan/piaociyuan-web` | `.github/workflows/deploy.yml` |
| 部署方式 | SSH → docker pull → docker-compose | `.github/workflows/deploy.yml` deploy job |

**需检查的文档**:
- `docs/项目架构与部署说明.md`
- `docs/阿里云部署指南.md`
- `CLAUDE.md`

## 6. 主题/配色一致性

Web 和 Mobile 的品牌色必须一致。

### 6.1 检查项

| 颜色 | 用途 | Web 位置 | Mobile 位置 |
|------|------|----------|-------------|
| `#46467A` | 主色调 (primary) | `apps/web/app/globals.css` → `--primary` | `apps/mobile/src/constants/config.ts` → `COLORS.primary` |
| `#E0DFFD` | 背景色 | `apps/web/styles/background.css` → `--app-bg-color` | `apps/mobile/src/constants/config.ts` → `COLORS.background` |
| `#1a1a1f` / `#282828` | 文字色 | `apps/web/app/globals.css` → `--foreground` | `apps/mobile/src/constants/config.ts` → `COLORS.text` |

> 注意: Web 文字色 (`#1a1a1f`) 与 Mobile 文字色 (`#282828`) 略有差异，属于已知差异。

**验证方法**:
```bash
grep -n "primary" apps/web/app/globals.css
grep -n "primary" apps/mobile/src/constants/config.ts
```

## 7. API URL 一致性

API 地址在多处定义，生产环境必须指向同一域名。

### 7.1 检查项

| 位置 | 文件 | 预期生产值 |
|------|------|------------|
| Mobile API 配置 | `apps/mobile/src/config/api.ts` | `EXPO_PUBLIC_API_URL` 环境变量 (fallback: `http://localhost:3000`) |
| Mobile 常量 | `apps/mobile/src/constants/config.ts` | `EXPO_PUBLIC_API_URL` (fallback: `https://piaociyuan.com`) |
| EAS Preview | `apps/mobile/eas.json` → preview | `https://piaociyuan.com` |
| EAS Production | `apps/mobile/eas.json` → production | `https://piaociyuan.com` |
| Web CORS | `apps/web/next.config.ts` | `CORS_ALLOWED_ORIGINS` (含 `https://piaociyuan.com`) |
| Socket.io CORS | `apps/web/server.js` | `NEXT_PUBLIC_APP_URL` |
| Docker Compose | `docker-compose.yml` | `NEXT_PUBLIC_APP_URL` (从 .env 读取) |
| 健康检查 | `.github/workflows/deploy.yml` | `https://piaociyuan.com/api/health` |

**验证方法**: 确认所有生产 URL 统一指向 `https://piaociyuan.com`。
