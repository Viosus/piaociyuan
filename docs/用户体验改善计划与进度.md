## 0. 用法说明

- **状态图例**：⬜ 未开始 / 🟡 进行中 / ✅ 已完成 / ⏸ 已搁置
- **优先级**：P0（阻塞日常使用）/ P1（高频痛点）/ P2（明显可感知但非必需）/ P3（锦上添花）
- **工作量**：S=半天内 / M=1-3 天 / L=1 周以上
- 每完成一项就在表格里更新状态、完成日期、关联 commit 或 PR
- 新增项追加到对应模块表底部；不要删除历史项

## 1. 全局原则

下面三条是写每一条改善前都应该自问的：

1. **延迟感受 < 100ms 才算"快"**：交互反馈（按下、loading、跳转）不能等服务器响应才出现；优先 optimistic update。
2. **网络不可靠是常态**：每个发请求的位置都要回答"如果失败/慢/超时，用户看到什么"。
3. **桌面与移动端是两种心智模型**：Web 强调键盘可达 + 信息密度；Mobile 强调单手可达 + 触感反馈。同一个功能两端都做，不代表 UI 也要长得一样。

| 私聊消息送达延迟（发送 → 对方端显示） | 双机实测 | 偶发 > 5s 或不送达 | P95 < 1s |

| 关键路径完成率（注册→发首贴） | 埋点 | 未埋 | > 30% |
