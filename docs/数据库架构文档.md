## 核心业务表

### 1. Event（活动表）
存储演出、展览等活动信息。

```prisma
model Event {
  id            Int      @id @default(autoincrement())
  name          String                    // 活动名称
  category      String   @default("concert")  // concert, festival, exhibition, musicale, show, sports, other
  city          String                    // 城市
  venue         String                    // 场馆
  date          String                    // 演出日期
  time          String                    // 演出时间
  saleStatus    String   @default("on_sale")  // not_started, on_sale, paused, sold_out, ended
  saleStartTime DateTime                  // 开售时间
  saleEndTime   DateTime                  // 停售时间
  cover         String                    // 封面图URL
  artist        String                    // 艺人/主办方
  desc          String                    // 活动描述
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  // 关联
  tiers         Tier[]
  collectibles  Collectible[]
  posts         Post[]
  followers     EventFollow[]
  notifications Notification[]
  homepageSections HomepageSectionEvent[]
}
```

**索引**: `category`, `date`

---

### 2. Tier（票档表）
定义活动的票价档位。

```prisma
model Tier {
  id        Int      @id @default(autoincrement())
  eventId   Int                           // 关联活动
  name      String                        // 票档名称（如"VIP看台"）
  price     Int                           // 单价（分）
  capacity  Int                           // 总容量
  remaining Int                           // 剩余数量
  sold      Int      @default(0)          // 已售数量
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  event     Event    @relation(...)
}
```

**索引**: `eventId`

---

### 3. Ticket（票表）
每张票都是独立实体，支持完整生命周期追踪。

```prisma
model Ticket {
  id          String   @id @default(uuid())
  ticketCode  String   @unique            // 票号（用于二维码）
  seatNumber  String?                     // 座位号
  eventId     Int
  tierId      Int
  holdId      String?                     // 锁票ID
  orderId     String?                     // 订单ID
  userId      String?                     // 用户ID
  status      String                      // available, locked, sold, used, refunded
  price       Int                         // 购买价格

  purchasedAt DateTime?                   // 购买时间
  usedAt      DateTime?                   // 核销时间
  refundedAt  DateTime?                   // 退款时间

  // 关联
  order   Order?   @relation(...)
  user    User?    @relation(...)
}
```

**票状态流转**:
```
available → locked → sold → used
                  ↓
              refunded → available
```

**索引**: `eventId`, `tierId`, `holdId`, `orderId`, `userId`, `status`, `ticketCode`, 复合索引 `[eventId, tierId, status, seatNumber]`

---

### 4. Order（订单表）
记录用户的购票订单。

```prisma
model Order {
  id              String   @id
  userId          String
  eventId         String
  tierId          String
  qty             Int                     // 购买数量
  holdId          String                  // 关联锁票
  status          String                  // pending, paid, cancelled, refunded
  createdAt       BigInt
  paidAt          BigInt?

  // 支付信息
  paymentMethod   String?                 // wechat, alipay, mock
  transactionId   String?                 // 第三方支付交易号
  refundedAt      BigInt?
  refundId        String?

  user      User     @relation(...)
  tickets   Ticket[]
}
```

**索引**: `userId`, `status`, `createdAt`, `transactionId`

---

### 5. Hold（锁票表）
临时锁定票务库存，防止超卖。

```prisma
model Hold {
  id        String   @id
  eventId   String
  tierId    String
  qty       Int                           // 锁定数量
  expireAt  BigInt                        // 过期时间戳
  createdAt BigInt
}
```

**索引**: `[eventId, tierId]`, `expireAt`

## 收藏品系统表

### 6. Collectible（收藏品定义表）
定义收藏品模板。

```prisma
model Collectible {
  id            String   @id @default(uuid())
  name          String
  description   String   @db.Text
  imageUrl      String

  // 分类
  category      String   // badge, ticket_stub, poster, certificate, art

  // 关联
  eventId       Int?
  tierId        Int?

  // 3D/AR 展示
  has3DModel    Boolean  @default(false)
  model3DUrl    String?
  modelFormat   String?
  hasAnimation  Boolean  @default(false)
  animationUrl  String?
  modelConfig   String?  @db.Text

  // 供应控制
  totalSupply   Int
  claimedCount  Int      @default(0)

  // 状态
  isActive      Boolean  @default(true)

  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

**索引**: `eventId`, `category`, `isActive`

## 高并发优化

### 防超卖机制
使用 PostgreSQL `FOR UPDATE SKIP LOCKED` 实现行级锁：

```sql
SELECT id, "ticketCode"
FROM tickets
WHERE "eventId" = $1 AND "tierId" = $2 AND status = 'available'
ORDER BY id
LIMIT $3
FOR UPDATE SKIP LOCKED
```

### 索引优化
- 所有状态字段建立索引
- 时间字段建立索引用于范围查询
- 高频组合查询使用复合索引

npx prisma migrate reset

npx prisma generate

# 打开数据库可视化工具
npx prisma studio
